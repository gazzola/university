Instruções de Submissão

Enviando Trabalhos de Programação

Cada trabalho tem um nome no sistema (p0, p1, etc). Para enviar um trabalho, compacte o diretório do projeto (por exemplo, tutorial) usando ZIP (por favor, não use RAR), e acrescente o último sobrenome de cada componente do grupo ao nome do arquivo, seguindo esse modelo: tutorial-SobrenomeSobrenome.zip. Envie o arquivo pelo Moodle, observando a data limite.

Uma submissão deve consistir de:

  • Um arquivo chamado grupo.txt. Esse arquivo deve conter uma lista dos participantes do grupo, incluindo quem está submetendo. Cada linha do arquivo deve estar no seguinte formato:

    [matricula],[email],[name]

    Por exemplo,

    111222333,john.mctest@gmail.com,John Q. McTest

    O endereço de email deve ser o cadastrado no Moodle. As notas serão calculadas automaticamente e submetidas para publicação no Moodle. Por isso, inclua esse arquivo mesmo que você tenha feito o trabalho sozinho.

  • Todos arquivos que tiverem sido editados. Em geral, serão fornecidos arquivos de suporte (código e/ou dados) para cada projeto, os quais você não deve alterar. O sistema de avaliação irá reclamar se isso acontecer.

Depois de encerrados os prazos para submissão, o sistema de avaliação será executado e irá realizar algumas verificações sobre sua submissão:

  • Um arquivo grupo.txt está presente;
  • Nenhum arquivo de suporte foi editado;
  • Seu código roda sem produzir nenhuma exceção;
  • Seu código passa por alguns casos de teste simples.

Tome cuidado ao submeter os projetos. Apenas a última submissão conta, e apenas um dos componentes do grupo deve fazer a submissão.

Enviando arquivos opcionais

Se seu grupo fizer algum dos problemas opcionais, não esqueça de incluir os arquivos necessários no pacote de submissão.
